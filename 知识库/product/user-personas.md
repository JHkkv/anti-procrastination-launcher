## 画像一：拖延大学生

### 基本信息

| 项目 | 详情 |
|------|------|
| **姓名** | 小林（化名） |
| **年龄** | 21 岁 |
| **身份** | 大三计算机专业学生 |
| **设备** | iPhone 15 + MacBook Air |
| **常用 App** | B站、微信、Notion（偶尔）、Forest（装了没怎么用） |
| **性格特征** | 完美主义倾向、焦虑型拖延、deadline驱动型 |

### 日常痛点

1. **"我要写论文"→ 打开电脑 → 刷B站3小时** —— 每次都这样，已经形成自动化行为了
2. **考试前一周爆发焦虑** —— 平时学不进去，考前通宵但效率极低
3. **任务太大不知道从哪开始** —— "完成课程设计"这个任务放在那里一周了
4. **自责循环** —— 拖延→自责→焦虑→继续拖延→更深的自责
5. **装了很多效率App但不用** —— Forest 树枯了几十棵，Notion 模板建了没打开过

### 使用场景

| 场景 | 时间 | 情绪状态 | 期望 |
|------|------|---------|------|
| 早上醒来准备学习 | 8:00-9:00 | 惰性、不想动 | 轻柔但坚定的启动引导 |
| 打开电脑后 | 9:00-9:30 | 无意识打开社交媒体 | 阻断+替代引导 |
| 作业 deadline 前一天 | 晚上 22:00 | 高度焦虑 | 把"写论文"拆成"打开文档写标题" |
| 考试周 | 全天 | 持续焦虑 | 每次只学25分钟，降低心理负担 |
| 深夜自责时 | 23:00-01:00 | 沮丧、自我否定 | 正向反馈、"今天做了X已经很好了" |

### 核心需求

- **P0**：能把"大作业"自动拆成"第一步做什么"
- **P0**：极低门槛——打开就看到下一步，不用配置
- **P1**：正向反馈而非惩罚（"你今天做了3步"而非"你又没完成"）
- **P1**：社交轻压力（室友也在用）
- **P2**：与校园日历/课程表整合

### 期望功能

1. 输入"完成数据库课程设计"→ AI 拆成 10 个小步骤
2. 每次只显示"下一步"，降低视觉压力
3. 完成一步后的微型庆祝动画
4. "2分钟模式"——先做2分钟，做完可以选择继续或停下
5. 每日回顾："你今天迈出了X步"（而非"你还有X项未完成"）

## 画像二：自由职业者

### 基本信息

| 项目 | 详情 |
|------|------|
| **姓名** | 阿晴（化名） |
| **年龄** | 28 岁 |
| **身份** | 自由插画师，接稿为主 |
| **设备** | iPad Pro + iPhone + 台式机（Win） |
| **常用 App** | Procreate、Pinterest、微信、淘宝、小红书 |
| **性格特征** | 高创造力、低结构化需求、情绪波动大 |

### 日常痛点

1. **没有外部结构** —— 不打卡上班，时间完全自己安排，结果每天睡到中午
2. **时间感知失真** —— "下午再画"→下午变成了"晚上再画"→"明天再画"
3. **多项目切换瘫痪** —— 同时有3个稿子，不知道先做哪个，结果哪个都没做
4. **收入不稳定带来的焦虑** —— 焦虑→刷手机逃避→更焦虑
5. **家里的干扰太多** —— 快递、猫、家务、洗衣机，每一个都比画画有吸引力

### 使用场景

| 场景 | 时间 | 情绪状态 | 期望 |
|------|------|---------|------|
| 起床后 | 10:00-11:00 | 迷茫、无目标感 | 今天最重要的3件事是什么 |
| 站在画板前 | 11:00-11:30 | 抗拒、找借口 | "先画5分钟线稿试试" |
| 客户催稿时 | 不定 | 紧张、自责 | 快速聚焦到最关键的部分 |
| 下午犯困时 | 14:00-15:00 | 疲惫、想躺平 | 短时冲刺+休息循环 |
| 晚上补作业时 | 22:00-02:00 | 疲惫但紧迫 | 分段推进，避免通宵 |

### 核心需求

- **P0**：每日优先级排序——今天最重要的3件事
- **P0**：极小步骤引导——"先打开Procreate"比"画完这张图"更可行
- **P1**：多项目管理——哪个项目该优先做
- **P1**：时间感知辅助——"你已经拖延了2小时"的温柔提醒
- **P2**：收入/进度关联——看到完成进度与收入的关系

### 期望功能

1. 每日晨间 check-in："今天有3个稿件，哪个deadline最近？"
2. AI 根据 deadline 自动排序优先级
3. 任务拆解到应用层："打开Procreate→打开昨天的草稿→画线稿"
4. "专注冲刺"模式——做25分钟+休息5分钟的极简循环
5. 每周回顾报告：完成了多少稿子，拖延了多少时间

## 画像三：职场新人

### 基本信息

| 项目 | 详情 |
|------|------|
| **姓名** | 小陈（化名） |
| **年龄** | 24 岁 |
| **身份** | 互联网公司产品运营，入职8个月 |
| **设备** | 公司 ThinkPad + 个人 iPhone |
| **常用 App** | 飞书、微信、小红书、滴答清单（装了） |
| **性格特征** | 社交焦虑、冒充者综合征、过度在意他人评价 |

### 日常痛点

1. **面对新任务的启动困难** —— "写一份竞品分析报告" → 不知道怎么开始 → 拖到deadline前赶
2. **会议后行动瘫痪** —— 会上记了 action items，会后不知道先做哪个
3. **冒充者综合征** —— 觉得自己不够好，不敢开始怕做错
4. **向上管理焦虑** —— 怕被领导发现进度慢，越怕越拖
5. **下班后反刍** —— 白天没做完的事，晚上焦虑但已经没精力做了

### 使用场景

| 场景 | 时间 | 情绪状态 | 期望 |
|------|------|---------|------|
| 早上到公司开电脑 | 9:00-9:30 | 困倦、不想面对 | 今天最重要的1件事是什么 |
| 收到新任务时 | 不定 | 恐慌、不知从何下手 | "第一步：花5分钟了解背景" |
| 午饭后犯困 | 13:30-14:00 | 疲惫 | 简单任务先行，降低启动难度 |
| 领导催进度时 | 不定 | 焦虑 | 快速聚焦、拆解到可执行 |
| 下班前 | 18:00-18:30 | 希望有点成就感 | "今天完成了X"的正向回顾 |

### 核心需求

- **P0**：任务拆解到"不用想"的程度
- **P0**：降低对"做不好"的恐惧——"先做，做完再改"
- **P1**：与工作工具（飞书/钉钉）的任务同步
- **P1**：情绪感知——焦虑时给更小的步骤
- **P2**：职场技能成长路径（不只是完成任务，还在变强）

### 期望功能

1. 粘贴任务描述 → AI 自动拆解为可执行步骤
2. "最小启动"模式——只做2分钟，降低心理门槛
3. 进度可视化——看得到自己在推进
4. 情绪check-in——"你现在感觉怎么样？"根据状态调整步骤大小
5. 每日结束时的正向回顾："今天迈出了X步"

## 画像四：考研/考公备考者

### 基本信息

| 项目 | 详情 |
|------|------|
| **姓名** | 小赵（化名） |
| **年龄** | 23 岁 |
| **身份** | 二战考研生 / 考公备考者 |
| **设备** | Android 手机 + 平板（用于看网课） |
| **常用 App** | B站（网课）、小红书（经验贴）、番茄TODO（装了但偶尔用）、微信 |
| **性格特征** | 目标明确但执行力不足、易受同辈压力影响、备考焦虑、长周期目标管理困难 |

### 日常痛点

1. **备考周期长导致动力衰减** —— 考研/考公备考通常6-12个月，前期热情高涨，中后期严重倦怠
2. **"今天该学什么"的每日决策** —— 科目多（政治、英语、专业课），每天选科目的决策本身就是消耗
3. **学习计划完不成的自责** —— 列了详细计划但执行不到50%，反复失败导致自我效能感崩塌
4. **同辈压力** —— 看到同学已经上岸/找到工作，自己还在备考，焦虑加倍
5. **信息过载** —— 经验贴太多，"该跟哪个老师""该用什么资料"的选择让人瘫痪
6. **网课依赖** —— 看网课感觉在学习，但实际吸收率很低，形成"伪学习"拖延

### 使用场景

| 场景 | 时间 | 情绪状态 | 期望 |
|------|------|---------|------|
| 早上到自习室 | 8:00-8:30 | 困倦、不想面对 | "今天最重要的一件事：背30个单词" |
| 刷完一节网课后 | 不定 | 疲惫、需要切换 | "接下来做5道对应的练习题" |
| 下午效率低谷 | 14:00-15:00 | 疲惫 | 简单任务先行（抄笔记/整理错题） |
| 晚上复盘时 | 21:00-22:00 | 可能自责 | "今天完成了4步，比昨天多1步" |
| 看到同学动态时 | 不定 | 焦虑、自我怀疑 | 温和引导回到自己的节奏 |

### 核心需求

- **P0**：每日学习计划拆解为具体可执行的微步骤（"背第3单元单词"而非"复习英语"）
- **P0**：科目轮换引导——避免一整天只学自己喜欢的科目
- **P1**：长周期进度可视化——看到自己在整个备考周期中的位置（对抗"还有好多没学"的焦虑）
- **P1**：正向回顾——"你本月完成了XX步"而非"距离考试还有XX天"（避免倒计时焦虑）
- **P2**：研友轻社交——"今天有XX人也在自习"的陪伴感

### 期望功能

1. 输入"复习高数第三章" → AI 拆解为"看知识点 → 做例题 → 做课后题5道"
2. 每日智能推荐：根据剩余时间和薄弱科目，推荐今天应该学什么
3. "先做5道题"模式——用微小承诺启动学习
4. 备考进度仪表盘——各科进度一目了然（但不显示倒计时）
5. 每周正向回顾："本周你迈出了32步，英语进步最明显"

## 画像五：轻度ADHD倾向者

### 基本信息

| 项目 | 详情 |
|------|------|
| **姓名** | 小王（化名） |
| **年龄** | 32 岁 |
| **身份** | 独立开发者/技术博主 |
| **设备** | 多设备（Mac/iPhone/Android测试机） |
| **常用 App** | Obsidian、GitHub、X(Twitter)、YouTube |
| **性格特征** | 高智商、高创造力、执行功能障碍、时间盲（time blindness） |

> 注：画像基于 ADHD 社区公开讨论，不代表临床诊断。
>
> **临床边界声明**：本画像描述的是"轻度ADHD倾向"的行为特征，而非确诊ADHD。ADHD（注意缺陷多动障碍）是一种神经发育障碍，需要由精神科医生或临床心理学家通过标准化评估工具（如DIVA 5.0、Conners量表）进行诊断。**如果用户怀疑自己可能患有ADHD，应首先寻求专业医疗评估，而非仅依赖自我管理工具。** 本应用可以作为辅助手段，但不能替代临床诊断和治疗（包括药物治疗和认知行为治疗）。

### 日常痛点

1. **执行功能障碍** —— 知道该做什么，但大脑拒绝启动
2. **时间盲（time blindness）** —— 完全没有时间感知，"5分钟"和"2小时"感觉一样
3. **超级聚焦 vs 完全停滞** —— 两极化状态，中间态很少
4. **任务优先级混乱** —— 所有事看起来一样重要/一样不重要
5. **工作记忆有限** —— 切换任务后忘了之前在做什么
6. **决策疲劳** —— "先做A还是B"这个问题本身消耗大量精力

### 使用场景

| 场景 | 时间 | 情绪状态 | 期望 |
|------|------|---------|------|
| 打开电脑后 | 不定 | 无方向感 | "你现在该做这件事"的明确指引 |
| 被打断后 | 不定 | 迷茫 | "回到刚才：你在写XX模块的第3步" |
| 进入超聚焦状态 | 不定 | 高度沉浸 | 不要打扰，但结束后提醒休息 |
| 同时想到5件事 | 不定 | 焦虑 | 快速脑倒（brain dump），AI 排序 |
| 傍晚能量低谷 | 16:00-18:00 | 低能量 | 只安排简单/机械性任务 |

### 核心需求

- **P0**：外部执行功能代理——替大脑决定"现在做什么"
- **P0**：零配置、零决策——打开就知道下一步
- **P0**：时间感知辅助——"你已经做了15分钟"的温柔提醒
- **P1**：脑倒功能——快速输入，AI 帮排序和拆解
- **P1**：被打断后快速恢复上下文
- **P2**：能量水平追踪——不同时段匹配不同难度的任务

### 期望功能

1. **"告诉我该做什么"模式** —— 打开App，直接显示当前应该做的唯一一件事
2. **时间可视化** —— 剩余时间的进度条（对抗时间盲）
3. **脑倒+AI排序** —— 随时输入想法，AI 自动归类到项目和优先级
4. **上下文快照** —— 被打断时保存当前步骤，回来时一键恢复
5. **不要惩罚机制** —— ADHD 用户对"失败"的敏感度更高，只用正向反馈
6. **极简UI** —— 屏幕上同时显示太多信息会触发决策瘫痪

## 关键洞察

### 四个画像的共同需求

1. **零配置** —— 不想花时间"设置工具"，只想用
2. **只看下一步** —— 完整任务列表让人窒息，只要当前这一步
3. **正向反馈** —— "你做到了"比"你还差X项"有效100倍
4. **AI拆解** —— 所有人都不想自己拆任务，这本身就是拖延源

### 差异化需求

| 需求 | 大学生 | 自由职业 | 职场新人 | 考研/考公 | ADHD |
|------|--------|---------|---------|----------|------|
| 社交问责 | 高 | 低 | 中 | 中 | 低 |
| 时间感知 | 低 | 中 | 低 | 中 | 极高 |
| 优先级排序 | 中 | 极高 | 高 | 高 | 高 |
| 情绪感知 | 高 | 中 | 高 | 中 | 中 |
| 上下文恢复 | 低 | 低 | 中 | 低 | 极高 |

### 产品设计启示

- **最小可用版本**应优先服务"任务拆解→只看下一步→正向反馈"这条核心链路
- **社交功能**可以作为P1，但要轻量化（不是组队打怪，是"朋友也在动"）
- **时间感知工具**对ADHD用户是刚需，但对其他用户是锦上添花
- **情绪check-in**能显著提升用户体验，但不是MVP必需
