# 🌸 拖延症启动器

> **不是帮你管理任务，而是帮你开始第一步。**

一个专为拖延症患者设计的单文件PWA应用。把大任务拆成极小的步骤，每次只显示一步，引导你"先动起来"。

**🎨 设计风格：温暖手帐风** — 霞鹜文楷书法字体 + 纸张纹理 + 便签纸卡片 + 印章按钮

**🔗 在线体验：** https://jhkkv.github.io/anti-procrastination-launcher/

---

## ✨ 核心理念

拖延的本质不是懒，而是**启动阻力太大**。

传统效率工具（Notion、滴答清单、Todoist）帮你管理任务列表，但对拖延症患者来说，看到长长的待办清单本身就是一种压力。拖延症启动器的思路完全不同：

- **只看下一步** -- 屏幕上永远只有当前这一小步
- **做比做好更重要** -- 没有截止日期，没有惩罚，只有温柔的鼓励
- **零配置启动** -- 打开就用，不需要花时间"设置工具"（那也是一种拖延）

## 🎯 功能特性

### 📋 预设流程模板

内置 5 个常见场景模板，开箱即用：

| 模板 | 适用场景 | 步骤数 |
|------|---------|--------|
| 🌅 早晨起床 | 从赖床到出门的完整流程 | 8 步 |
| 📚 学习启动 | 从"不想学"到进入状态 | 6 步 |
| 💻 工作开工 | 打开电脑到进入专注 | 7 步 |
| 🌙 睡前准备 | 从刷手机到入睡 | 6 步 |
| 🧹 家务启动 | 从"不想动"到开始收拾 | 5 步 |

### 👣 一步一动模式

屏幕只显示当前要做的一步。完成后点击"做到了"，自动切换到下一步。你永远不需要面对"还有 20 步"的压力。

### 💬 温柔鼓励系统

每完成一步，随机弹出一句治愈系鼓励语。没有惩罚机制，只有正向反馈：

- "很棒，你又迈出了一小步 ✨"
- "做到了就是做到了，不需要更多理由 🌱"
- "你看，开始之后也没那么难吧 🌸"

### ✏️ 自定义流程

支持创建自己的流程模板，适合有固定日程的场景。

### 📊 进度追踪

记录每天完成了多少步，以温柔的方式回顾你的进步。

### 📱 PWA 离线可用

支持安装到手机主屏幕，无需网络即可使用。

## 📖 使用指南

### 使用预设模板

1. 打开应用，首页展示所有预设模板
2. 点击一个模板（如"🌅 早晨起床"）
3. 屏幕显示第一步（如"深呼吸三次"）
4. 做完后点击"做到了 ✨"
5. 自动进入下一步，直到全部完成

### 创建自定义流程

1. 点击首页的"+"按钮
2. 输入流程名称（如"画画启动"）
3. 逐步添加小步骤（建议 5-8 步，每步不超过 2 分钟）
4. 保存后即可使用

### 查看进度

- 首页底部展示今日完成步数
- 点击可查看历史记录
- 正向回顾：只看"做到了多少"，不看"还差多少"

## 🎨 设计理念

### 日系治愈风

视觉风格参考日本治愈系App，强调温暖、柔和、无压力的视觉感受：

- **色彩**：樱花粉 `#FFB7C5` 为主色，暖白 `#FFF8F0` 为底色
- **圆角**：大面积圆角设计，消除锐利感
- **动画**：缓慢渐入渐出，模拟呼吸节奏
- **字体**：圆润的无衬线字体，降低阅读压力

### 认知负荷最低化

- 一次只展示一个信息单元
- 按钮文案用行动导向语言（"做到了"而非"完成"）
- 隐藏所有非必要元素
- 无数字、无列表、无时间压力

### 正向反馈机制

基于拖延心理学研究（自我决定理论），设计原则：

- **只有奖励，没有惩罚** -- 完成了有鼓励，没完成没有任何负面反馈
- **关注过程，而非结果** -- "你迈出了这一步"比"你完成了任务"更有激励效果
- **降低期望门槛** -- 每一步都设计为 1-2 分钟内可完成

### 无压力设计

- 没有截止日期提醒
- 没有连续天数统计（断签焦虑）
- 没有与他人的比较
- 没有"未完成"的视觉提示

## 🛠️ 开发信息

### 技术栈

| 技术 | 说明 |
|------|------|
| HTML5 | 语义化结构 |
| CSS3 | CSS 变量 + Flexbox/Grid 布局 |
| Vanilla JavaScript | 零框架依赖，全内联实现 |
| PWA | Service Worker + Web App Manifest，支持离线和安装 |

**设计决策**：选择单文件全内联方案，而非模块化拆分，原因是：
- 零构建工具，降低维护门槛
- 单文件部署，任何静态服务器都能跑
- 极小的文件体积（< 50KB）

### 多 Agent 开发流程

本项目采用 9 个专职 Agent 协作开发：

| Agent | 职责 |
|-------|------|
| Orchestrator | 总调度，任务分派与结果汇总 |
| 心理学研究员 | 构建拖延心理学 RAG 知识库 |
| UI/UX 研究员 | 日系治愈风设计规范研究 |
| 竞品分析师 | 市场调研与用户画像 |
| 前端开发者 | HTML+CSS+JS 核心代码实现 |
| 对抗性审查员 | 代码与设计质量验证 |
| 交互设计师 | 微动画与治愈风 UI 打磨 |
| 测试工程师 | 功能测试与无障碍验证 |
| 财务 Agent | Token 消耗优化策略 |

### Token 优化策略摘要

通过 5 项优化策略，将多 Agent 开发的总成本降低了 **39.8%**（从 $9.98 降至 $6.01）：

1. **CLAUDE.md 精简** -- 按 Agent 角色注入最小必要规则子集
2. **任务上下文压缩** -- 结构化摘要替代全文传递
3. **知识库模板复用** -- 基于模板填充而非从零生成
4. **输出格式精简** -- 结构化 schema 替代长段叙述（节约最大，占 41.6%）
5. **Prompt Cache 前缀最大化** -- 共享可缓存前缀，后续调用按 Cache Read 计费

详见 [财务报告-Token优化策略.md](财务报告-Token优化策略.md)。
